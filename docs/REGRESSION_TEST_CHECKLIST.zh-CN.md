# yonxao-mindmap 人工回归测试清单

这份清单用于重构前后做人工回归。测试样例主要放在
`./regression-layout-gallery.zh-CN.md`，建议把该文件内容放到 Obsidian 测试库中，
分别在 Live Preview 和阅读视图中检查。

## 测试原则

- 每次重构阶段完成后，至少跑一遍本清单中的核心路径。
- 布局、视口、导出、保存相关改动完成后，需要跑完整清单。
- 若发现视觉问题，记录布局类型、视图模式、主题方案、代码块序号和截图。
- 插件尚未发布，不检查旧配置兼容；只检查当前配置结构。

## 0. 基础环境

- 打开 Obsidian 测试库。
- 安装当前构建产物。
- 准备一篇包含 `examples/regression-layout-gallery.zh-CN.md` 内容的测试笔记。
- 准备浅色主题和深色主题各测试一次。
- 分别测试 Live Preview 和阅读视图。
- 如果继续保持 `manifest.json` 中 `"isDesktopOnly": false`，移动端至少检查“能正常显示、不崩溃、不可用能力有合理降级”。

## 1. 全部布局矩阵

对下列每个布局，都检查小图、中图、长文本图：

- `mindmap-right`
- `mindmap-left`
- `mindmap-bidirectional`
- `mindmap-down`
- `mindmap-up`
- `mindmap-vertical`
- `tree-right`
- `tree-left`
- `tree`
- `org`
- `org-right`
- `timeline-up`
- `timeline-down`
- `timeline`
- `radial`
- `fishbone-left`
- `fishbone-right`
- `tree-table`
- `tree-table-stepped`

检查项：

- 主题不重叠。
- 连线不穿过主题文本。
- 折叠按钮位于当前主题的子线出口附近。
- 新增按钮、编辑按钮位置合理。
- 长文本主题不溢出主题框。
- 多行主题按真实换行显示。
- 主题最大宽度生效。
- 自动配色和主题属性颜色都正确。

## 2. 视图模式

对每个布局至少抽查一个中图和一个长文本图：

- 打开后默认 `适配视图`，导图完整显示。
- 点击工具栏“原始大小”，主题字号不被自动缩放。
- 原始大小下，中心主题或结构起点位置合理，不被简单裁切到边缘。
- 再点击“适配视图”，导图恢复完整显示。
- 自动高度下，幕布高度能容纳完整导图。
- 手动拖动幕布高度后，配置区写入 `display.canvasHeight`。
- 双击高度拖拽条后，恢复自动高度并删除对应配置项。
- 全屏查看能进入和退出，工具栏位置合理。

## 3. 连线线型与下挂展开

使用样例中的“连线线型与下挂展开专项”：

- `curve` 显示曲线。
- `straight` 显示直线。
- `elbow` 显示折线。
- 思维导图中，只有 `connectorStyle: elbow` 时 `branchExpansion: hanging` 生效。
- 非思维导图布局中，连线固定为折线。
- 放射图和树形表格不支持下挂展开，配置面板中应显示不可用状态。

## 4. 多代码块场景

使用样例中的“多代码块专项”：

- 同一篇笔记中多个 `yxmm` 代码块都能独立渲染。
- 在第一个代码块中修改配置，只保存到第一个代码块。
- 在第二个代码块中修改配置，只保存到第二个代码块。
- 对第一个代码块复制图片后，再对第二个代码块复制图片，剪贴板内容应切换为第二个代码块。
- 导出图片时，文件内容对应当前操作的代码块。
- 源码模式保存时，只替换当前代码块，不误改其他代码块。

## 5. 主题编辑

使用样例中的“主题编辑专项”：

- 打开主题编辑面板，面板浮在当前主题附近。
- 面板可拖动，位置不会被幕布裁切。
- 修改内容后保存，源码同步更新。
- 输入多行内容后保存，正文区使用普通文本行保存多行内容。
- 修改颜色、图标、字体、字号、字重、行高、最大宽度后保存，主题显示正确。
- 点击取消后，内容和属性都恢复原值。
- 删除主题时，带子主题的主题需要确认。
- 新增子主题后，主题级别和源码位置正确。
- 双击主题进入快速编辑，文本框不随导图缩放异常放大或缩小。

## 6. 配置面板

分别在插件全局默认值配置和单个代码块配置中测试：

- 全局默认值能正确回填。
- 代码块配置能覆盖全局默认值。
- 主题属性能覆盖代码块配置。
- 回填值显示为默认值样式，自定义值显示为普通样式。
- 配置项改回当前全局默认值后，代码块配置区删除该项。
- 保存后配置区保持简洁，不写入无意义空对象。
- 字体下拉框能正确显示预设值。
- 自定义字体输入为空时不显示 placeholder 或伪值。
- 最大主题宽度范围提示正确，超出范围时被规范化。
- 高级配置页 YAML 与可视化配置页保持一致。

## 7. 源码模式

- 配置区和正文区为两个选项卡。
- 正文区主题级别高亮清晰。
- 普通文本行并入最近主题作为多行主题内容。
- 单行两个空格不再被当成换行语义。
- `Tab` / `Shift+Tab` 调整主题级别遵循配置。
- 源码实时同步到当前 Markdown 代码块。
- 源码解析错误时显示提示，不写入破坏性内容。

## 8. 右键菜单、复制和导出

- 空白处右键菜单可用。
- 主题右键菜单可用。
- “复制正文”会剔除主题属性。
- “复制正文 - # 替换为缩进”结构正确。
- “复制子树”只复制当前主题及后代。
- “复制子树 - # 替换为缩进”结构正确。
- “导出图片”导出当前导图。
- “复制图片”复制当前导图，而不是上一次操作的代码块。

## 9. 移动端降级

如果继续声明非桌面专用：

- 阅读视图能显示导图。
- 基础缩放或滚动不阻塞页面。
- 不支持的剪贴板、拖拽、全屏能力不应导致报错中断。
- 工具栏不会遮挡主要内容到无法操作。
