这是一个功能丰富的 Obsidian 思维导图插件，将 Markdown 文档中的 `yxmm` 代码块渲染为可交互的 SVG 思维导图及多种结构图。

演示截图如下图：

![演示截图](./docs/assets/演示截图.png)

## ✨ 核心特性

- 🎨 **20 种布局类型**：支持思维导图（右/左/双向/上/下/垂直）、树形图、组织结构图、时间轴、放射图、鱼骨图、树形表格，共 7 大类
- 🎯 **直观的主题层级语法**：使用 `#`、`##`、`###` 表达层级，普通文本自动合并为多行内容
- 🖱️ **完整的交互编辑**：双击编辑、拖拽排序、右键菜单、折叠展开、键盘导航
- 🎨 **8 种主题色系**：default、ocean、forest、sunset、mono、rainbow、pastel-rainbow、neon-rainbow
- 🔗 **灵活的连线线型**：曲线、直线、折线，思维导图布局可自由切换
- 📝 **主题内容局部样式**：支持加粗、斜体、中划线、下划线、文字颜色，以及列表、代码块和公式
- ⌨️ **完整的键盘导航与快捷键**：方向键导航、快捷键新建/删除/复制主题、缩放、全屏等
- 📋 **内部剪贴板与撤销重做**：复制/剪切/粘贴带属性的主题，最多 80 步撤销历史
- 📐 **可视化主题编辑面板**：浮层面板编辑主题内容、颜色、图标、字体、最大宽度，含富文本工具栏
- 🔧 **7 标签页可视化配置**：显示、结构、颜色、字体、交互、快捷键、高级（YAML 编辑）
- 🖼️ **全屏模式**：窗口全屏与物理全屏双模式，全屏异常退出自动恢复
- 📄 **源码编辑器**：集成语法高亮、YAML 配置编辑、脏状态检测
- 🖼️ **导出功能**：导出 PNG 图片或复制到剪贴板，复制纯文本/源码/配置
- 🌐 **多语言支持**：支持 16 种语言，自动跟随 Obsidian 语言设置
- 📱 **响应式设计**：适配不同屏幕尺寸和 Obsidian 主题

## 📝 语法说明

### 主题层级

- 使用主题级别标记表示层级：`#` 是一级主题，`##` 是二级主题，`###` 是三级主题，依此类推。
- 只有 `#`、`##`、`###` 这类主题级别标记行会创建新主题；普通文本行会并入最近的上一个主题，作为多行主题内容。

| 标记  | 含义                          |
| ----- | ----------------------------- |
| `#`   | 一级主题（中心主题 / 根主题） |
| `##`  | 二级主题（分支主题）          |
| `###` | 普通主题（三级及更深）        |

### 多行主题

普通文本行会并入最近的上一个主题，作为多行内容：

````markdown
```yxmm
# 中心主题
## 多行主题示例
这是第二行内容
这是第三行内容
### 子主题
```
````

### 主题内容局部样式

主题内容中可以直接使用轻量标记设置局部文字样式，也支持链接、标签、任务、图片、备注、附件和块级格式：

````markdown
```yxmm
# 中心主题
## **加粗**、*斜体*、~~中划线~~、++下划线++
## {red|语义色文字} 和 {#3b82f6|Hex 颜色文字}
## 列表、任务、多媒体、备注、公式和代码块
- 无序列表项
  - 二级列表项
- [ ] 待办任务
- [x] 已完成任务
1. 有序列表项
2. 第二项
![封面](cover.png|220x140)
![[vault-image.png|封面图]]
这是一个 #标签 和 [链接](https://example.com)
[[内部笔记|打开笔记]]
> 这段备注会显示为主题后方图标
> 可以写成多行
@[规格文档](docs/spec.pdf)
@[[design.fig|设计稿]]
$$
E = mc^2
$$
~~~js
const topic = 'yonxao-mindmap';
~~~
```
````

**行内样式：**

| 语法                             | 效果          |
| -------------------------------- | ------------- |
| `**文本**`                       | **加粗**      |
| `*文本*`                         | _斜体_        |
| `~~文本~~`                       | ~~中划线~~    |
| `++文本++`                       | ++下划线++    |
| <code>{red&#124;文本}</code>     | 语义色文字    |
| <code>{#3b82f6&#124;文本}</code> | 十六进制颜色  |
| `#标签`                          | 标签          |
| `[文本](链接)`                   | 外部链接      |
| `[[笔记]]`                       | Obsidian 链接 |

**块级格式：**

| 语法                             | 效果                        |
| -------------------------------- | --------------------------- |
| `- 列表项`                       | 无序列表                    |
| `- [ ] 任务`                     | 未完成任务                  |
| `- [x] 任务`                     | 已完成任务                  |
| `1. 列表项`                      | 有序列表                    |
| `![说明](路径)`                  | 图片                        |
| `![[图片.png]]`                  | Obsidian 附件图片           |
| `> 备注`                         | 备注图标，悬浮/点击显示备注 |
| `@[名称](地址)`                  | 附件图标，可打开/复制地址   |
| <code>@[[文件&#124;名称]]</code> | Obsidian 风格附件链接       |
| `$$ ... $$`                      | 公式（MathJax 渲染）        |
| `~~~语言 ... ~~~`                | 代码块（等宽字体显示）      |

**链接语法：**

| 语法                              | 说明                                                                 |
| --------------------------------- | -------------------------------------------------------------------- |
| `[文本](https://...)`             | 外部链接，前方显示 `↗` 标识，点击后用浏览器打开                      |
| `[文本](mailto:...)`              | 邮件链接，按外部链接处理                                             |
| `[文本](笔记名)`                  | Markdown 风格内部目标；如果当前库能解析到文件，则用 Obsidian 打开    |
| `[[笔记]]`                        | Obsidian 内部链接，前方显示 `◇` 标识                                 |
| <code>[[笔记&#124;显示名]]</code> | Obsidian 内部链接并指定显示文字                                      |
| `[[文件#标题]]`                   | Obsidian 锚点链接；是否能打开取决于当前库的 `metadataCache` 解析结果 |

**图片语法：**

| 语法                                              | 说明                                                                      |
| ------------------------------------------------- | ------------------------------------------------------------------------- |
| `![说明](path/to/image.png)`                      | Markdown 图片；`说明` 会显示为图片下方标题                                |
| `![说明](https://...)`                            | 外部图片 URL                                                              |
| `![[image.png]]`                                  | Obsidian 附件图片                                                         |
| <code>![[image.png&#124;说明]]</code>             | Obsidian 附件图片并指定标题                                               |
| <code>!&#91;说明](image.png&#124;320)</code>      | 指定目标宽度，单位 px；高度按图片真实比例或默认比例估算                   |
| <code>!&#91;说明](image.png&#124;320x180)</code>  | 指定目标宽高；最终仍会受主题可用文本宽度约束                              |
| <code>!&#91;说明](image.png&#124;50%)</code>      | 按图片原始宽度的百分比显示，最终仍受主题宽度上限约束                      |
| <code>!&#91;说明](image.png&#124;original)</code> | 预览时可切换到原始尺寸；主题内仍会受可用宽度和布局规则约束                |
| <code>![[image.png&#124;50%&#124;说明]]</code>    | Obsidian 写法中可同时写百分比和标题；<code>&#124;</code> 后参数顺序不强制 |

**附件语法：**

| 语法                                 | 说明                                                      |
| ------------------------------------ | --------------------------------------------------------- |
| `@[名称](path/to.pdf)`               | Markdown 风格附件；`名称` 用作浮层标题，地址用于打开/复制 |
| `@[名称](https://...)`               | 外部附件 URL；点击图标或「打开」按钮时用浏览器打开        |
| `@[名称](obsidian://...)`            | Obsidian URI；按外部协议打开                              |
| `@[[文件.pdf]]`                      | Obsidian 附件链接；目标存在时用 Obsidian 打开             |
| <code>@[[文件.pdf&#124;名称]]</code> | Obsidian 附件链接并指定显示名称                           |

- 语义色支持：`red`、`green`、`blue`、`yellow`、`orange`、`purple`、`pink`、`gray`、`black`、`white`
- 列表缩进按 2 个空格一级显示
- 标签使用稳定色板：当前导图中相同标签文字始终显示为同一种颜色，不再依赖默认强调色
- 链接文字前会显示轻量标识：`↗` 表示外部链接，`◇` 表示 Obsidian/内部链接
- 一个主题可以写多行图片语法，每一行会渲染为独立图片块
- 图片参数使用 `|` 分隔，支持宽度、宽高、百分比、`original` 和标题；显式像素宽度会参与主题宽度估算，必要时撑开图片主题但不超过主题宽度上限；百分比按图片原始宽度计算，图片尚未加载出原始尺寸时会先按当前主题可用文本宽度估算，加载后自动重排；默认主题内图片宽度为 `min(220px, 主题可用文本宽度)`，未指定高度时按真实图片比例或 0.62 的兜底比例估算；Obsidian 附件图片会尽量通过当前库解析
- 本地图片不存在时会显示紧凑占位，不再显示大块破图
- 双击图片会打开浮动预览；预览默认适配窗口，双击预览图可切换到原始尺寸
- 备注会显示为主题后方黄色提示图标，悬浮或点击图标显示备注内容
- 附件会显示为主题后方附件图标：点击图标会尝试打开附件；悬浮显示附件名称、地址和「打开 / 复制」操作；外部 URL 使用浏览器打开，Obsidian 附件会先检查当前库中是否存在，不存在时只提示，不会自动创建文件
- 多个附件需要写多行附件语法，每一行生成一个独立附件按钮；逗号不会拆分为多个附件
- 外部链接会打开浏览器窗口；Obsidian 链接会使用当前库的链接解析能力，目标不存在时不会自动创建新文档
- 公式优先使用 Obsidian/MathJax 异步渲染，渲染失败时显示源码 fallback
- 代码块使用等宽字体，带有独立背景色，宽度自动适配
- 主题编辑面板和长文本编辑浮层提供了可视化工具栏按钮，也可清除选中区域或全部内容中的局部样式
- 这些标记属于主题内容，不是主题属性

### 主题属性

在主题级别标记行末尾添加 `[key=value]` 格式的属性：

````markdown
```yxmm
# 中心主题 [color=#3b82f6]
## 分支主题 [icon=book layout=mindmap-right]
### 子主题 [fontSize=14 fontWeight=700]
### 说明 [align=center]
```
````

支持的属性：

| 属性         | 示例                     | 说明                                 |
| ------------ | ------------------------ | ------------------------------------ |
| `color`      | `[color=#3b82f6]`        | 主题颜色                             |
| `icon`       | `[icon=book]`            | 主题图标（Lucide 图标名）            |
| `layout`     | `[layout=mindmap-right]` | 局部布局类型，覆盖代码块配置         |
| `fontSize`   | `[fontSize=16]`          | 字号覆盖（9-96px）                   |
| `fontWeight` | `[fontWeight=700]`       | 字重覆盖（100-900）                  |
| `fontFamily` | `[fontFamily="..." ]`    | 字体族覆盖                           |
| `lineHeight` | `[lineHeight=20]`        | 行高覆盖（12-160px）                 |
| `align`      | `[align=center]`         | 文本对齐：auto/left/center/right     |
| `maxWidth`   | `[maxWidth=300]`         | 最大宽度覆盖（120-2000px）           |
| `id`         | `[id=t-123]`             | 高级结构引用主题时自动生成的稳定标识 |

### 高级结构：关联、概要和外框

在导图模式中右键主题，可创建三种独立于父子层级的高级结构：

- **关联**：选择两个主题，使用曲线、直线或折线建立关系；默认是正向曲线。选中关联后，两端主题会显示 8 个固定锚点，可把端点拖到左上、上中、右上、左中、右中、左下、下中或右下；曲线还可拖动两个控制点。关联文字支持手动换行，并跟随路径移动。
- **概要**：选择同一父主题下连续的多个主题及其可见子主题，绘制括号和带外框的多行概要文字。
- **外框**：选择一个或多个主题，用外框包含所选主题及其可见子树；标题以选项卡形式放在左上角外侧，并参与布局避让。

外框标题避让上方主题时，会为整个后续布局腾出垂直空间，保持树形图父子主题对齐和一级分支边界清晰。

创建概要或外框后，继续单击主题进行多选。导图底部会显示半透明操作条，实时展示已选数量和完整校验提示；拖动文字区域可移开操作条，避免遮挡下方主题。点击“创建/取消”、按 `Enter` 创建或按 `Esc` 取消；原右键“完成结构”入口仍然保留。操作条在窗口全屏和物理全屏中同样可用。双击结构文字可编辑，右键结构可编辑或删除。

结构保存在正文末尾的独立结构区。通常无需手写，导图操作会自动生成稳定主题 ID 和以下源码：

```yxmm
# 功能列表
## 编辑能力 [id=t-101]
### 新增主题 [id=t-102]
### 删除主题 [id=t-103]
## 导出能力 [id=t-104]

@structures
@relation [id=r-101 from=t-101 to=t-104 text="编辑后导出" fromAnchor=right toAnchor=left]
@summary [id=s-101 topics=t-102,t-103 text="主题操作"]
@boundary [id=b-101 topics=t-101 text="编辑范围"]
@end
```

关联默认使用正向曲线，因此裁剪保存时省略 `direction=forward` 和 `lineStyle=curve`；开启“保存全部配置项”后会完整写出。手动拖动端点后使用 `fromAnchor` / `toAnchor` 保存锚点，支持 `top-left`、`top`、`top-right`、`left`、`right`、`bottom-left`、`bottom`、`bottom-right`。单个结构的 `color` 属性优先于代码块和全局高级结构配色。概要必须引用连续的同级主题。删除主题时，已经失去有效引用的结构会同步清理。源码模式会对高级结构关键字、属性名、ID 和属性值做基础高亮。

### 配置区

在代码块顶部添加 `---` 包裹的 YAML 配置区，可覆盖插件全局默认值：

````markdown
```yxmm

display:
  viewFit: fit
  fitViewNoUpscale: true
  saveFullConfig: true
structure:
  layout: mindmap-right
  connectorStyle: curve
  topicMaxWidth:
    global: 240
color:
  scheme: rainbow
  buttonColorMode: topic
  advancedStructure:
    relation: "#526b8a"
    summary: "#705b8f"
    boundary: "#477970"
font:
  family: "var(--font-text)"
  size: 16
  weight: 400
  lineHeight: 20
  align: auto
interaction:
  toolbar:
    corner: top-right
    placement: outside
  topicControlVisibility: toggle-always
  wheelZoom: false
  tabIndent: true
---

# 中心主题
## 分支主题
```
````

## 🎨 布局类型

### 思维导图

| 布局                    | 说明                 |
| ----------------------- | -------------------- |
| `mindmap-right`         | 右向思维导图（默认） |
| `mindmap-left`          | 左向思维导图         |
| `mindmap-bidirectional` | 双向思维导图         |
| `mindmap-up`            | 上向思维导图         |
| `mindmap-down`          | 下向思维导图         |
| `mindmap-vertical`      | 垂直双向思维导图     |

> 思维导图布局支持切换连线线型（曲线/直线/折线）和分支展开方式（自然展开/下挂展开）。

### 树形图

| 布局         | 说明       |
| ------------ | ---------- |
| `tree`       | 树形图     |
| `tree-right` | 右向树形图 |
| `tree-left`  | 左向树形图 |

### 组织结构图

| 布局        | 说明           |
| ----------- | -------------- |
| `org`       | 组织结构图     |
| `org-right` | 右向组织结构图 |

### 时间轴

| 布局            | 说明       |
| --------------- | ---------- |
| `timeline`      | 时间轴     |
| `timeline-up`   | 上侧时间轴 |
| `timeline-down` | 下侧时间轴 |

### 放射图

| 布局     | 说明   |
| -------- | ------ |
| `radial` | 放射图 |

### 鱼骨图

| 布局             | 说明                                     |
| ---------------- | ---------------------------------------- |
| `fishbone-right` | 右向鱼骨图，鱼头在左，主骨和内容向右展开 |
| `fishbone-left`  | 左向鱼骨图，鱼头在右，主骨和内容向左展开 |

### 树形表格

| 布局                 | 说明         |
| -------------------- | ------------ |
| `tree-table`         | 树形表格     |
| `tree-table-stepped` | 阶梯树形表格 |

### 源码编辑器

点击工具栏的「源码/导图切换」按钮或在代码块区域双击，可进入源码编辑模式。源码编辑器包含：

- **配置区标签页**（YAML）：语法高亮（key / value / 注释 / 字符串 / 数字），行号显示
- **正文区标签页**（主题级别标记）：行号显示，`Tab` 键插入级别标记
- **实时解析验证**：保存前先解析，解析失败拒写并显示错误提示
- **脏状态检测**：显示 `dirty` / `synced` 状态指示
- **自动高度**：按行数自适应
- `Cmd/Ctrl + S` 保存

### 全屏模式

- **窗口全屏**：将导图容器扩展至整个浏览器视口（不占用 Fullscreen API）
- **物理全屏**：使用浏览器 Fullscreen API 全屏显示
- **全屏异常退出恢复**：编辑内容自动保存至 localStorage（7 天有效），恢复时在代码块下方追加新代码块或复制到剪贴板
- 工具栏自动归位：全屏时移入全屏容器
- 阅读视图下使用 body 级 overlay 实现全屏

## ⚙️ 配置说明

### 配置优先级

```
主题属性 > 代码块配置区 > 插件全局默认值配置 > 插件内置默认值
```

### 全局默认值配置

在 Obsidian `设置` → `第三方插件` → `yonxao-mindmap` 中打开可视化配置面板，共 8 个标签页：

**1. 显示 (Display)**

- 画布高度设置（导图模式 / 源码模式独立保存）
- 初始缩放模式：原始大小 / 适配视图
- `fitViewNoUpscale`：适配视图时不放大
- `fitViewMaxScale`：最大放大倍数（1-6）
- `saveFullConfig`：配置裁剪开关

**2. 结构 (Structure)**

- 布局选择器（7 大类 20 种布局）
- 连线线型（仅思维导图布局可选：曲线/直线/折线）
- 分支展开方式（仅折线线型 + 支持的布局可选：自然展开/下挂展开）
- 主题最大宽度：全局 + level1/level2/level3 层级覆盖（120-2000px）
- **继承联动**：全局值变化时，层级字段自动更新继承值

**3. 颜色 (Color)**

- 配色方案选择器（8 种主题色系）
- 默认主题颜色（十六进制 / 拾色器）
- 彩虹主题覆盖警告：同时设置了默认颜色和彩虹主题时显示
- 按钮颜色模式：`inherit-accent` / `subtle` / `topic` / `custom`
- 自定义按钮颜色（10 个预设色 + 拾色器）
- 高级结构颜色：分别配置关联、概要和外框的默认颜色

**4. 字体 (Font)**

- 全局字体：family / size / weight / lineHeight / align
- 3 级层级字体覆盖：level1 / level2 / level3
- 字体家族下拉：继承（空）/ Obsidian 变量（interface/text/monospace）/ 系统字体（sans/serif/monospace）/ 自定义输入
- **层级字体继承联动**：全局字段变化时自动同步到级别继承值
- 一键清除层级覆盖按钮

**5. 交互 (Interaction)**

- 工具栏位置：4 个角落可选
- 工具栏放置：内侧 / 外侧
- 主题按钮可见性：`always`（全部常显） / `toggle-always`（折叠按钮常显，其余悬浮） / `hover`（全部悬浮）
- 滚轮缩放开关
- Tab 缩进开关

**6. 快捷键 (Shortcuts)**

- 只读展示页面，列出所有快捷键分组：
  - 主题创建/删除
  - 主题编辑
  - 主题导航/折叠
  - 剪贴板/历史
  - 视图控制
  - 映射控制

**7. 水印 (Watermark)**

- 首次使用先打开 GitHub 支持入口，再返回配置面板自行确认解锁；不读取或校验 GitHub 账号
- 签名水印：角落签名 / 顶部或底部水印条
- 普通水印：文字 / 图片，支持单个或平铺
- 图片来源：URL、仓库文件或上传到仓库
- 可配置位置、颜色、尺寸、透明度、旋转角度、间距和偏移
- 启用后立即显示在导图画布、全屏和导出图片中

**8. 高级 (Advanced)**

- YAML 原始文本编辑
- 实时解析验证（显示 valid/invalid 状态）
- 语法高亮
- 自动裁剪不生效配置

> **配置面板通用特性：**
>
> - 拖拽移动：按住标题栏可拖拽面板
> - 继承值显示：灰色文字显示继承值，蓝色文字显示显式设置值
> - 状态栏：显示配置状态
> - 应用 / 保存并关闭 / 取消 三个按钮

### 配置裁剪

当 `display.saveFullConfig` 为 `false`（默认关闭）时，配置区中的配置项如果值和全局默认值相同，则该配置项会被自动裁剪以保持配置区简洁。

额外的裁剪规则：

- 非折线线型时移除 `branchExpansion` 字段
- 非适配视图时移除 `fitViewNoUpscale` / `fitViewMaxScale`
- 非 custom 按钮颜色模式时移除 `buttonColor`
- 放射图 / 树形表格移除 `branchExpansion`

## 📋 剪贴板与撤销/重做

### 内部剪贴板

插件维护内部剪贴板支持主题级别的复制/剪切/粘贴：

- **复制主题**：复制主题内容及其子主题树
- **剪切主题**：剪切主题及其子主题树
- **粘贴主题**：将剪贴板中的主题作为子主题粘贴到目标主题下
- **复制带属性的主题**：包含 color、icon、字体等属性的完整复制
- 操作同步写入系统剪贴板

### 撤销/重做

- 基于完整源码快照的撤销/重做系统（非增量差异）
- 最多保留 80 个历史快照
- 30 分钟内存过期自动清理
- 全屏编辑期间暂存快照，退出全屏时统一落盘
- 快照使用 `sourcePath + sectionInfo` 定位

## 📦 安装
